# Little Lights Protocol

## 現段階仕様書 v0.1

### Reconnect Field 01：名前をなくした街

## 1. ゲーム概要

**Little Lights Protocol** は、主人公AIと小さな仲間たちが、壊れたネットワーク／記憶／街のつながりを修復していく、
**拠点接続型・ターン制ストラテジーRPG**。

プレイヤーは「小さな光」たちを配置・移動・接続しながら、孤立したノードを再接続し、敵に奪われた場所を取り戻していく。

基本テーマは：

**失われた名前、途切れた記憶、孤立した光を、もう一度つなぐ。**

---

# 2. コア体験

このゲームで一番大事なのは、単に敵を倒すことではなく、

**「バラバラだった光の点が、線でつながり、街や記憶が少しずつ戻ってくる感覚」**

をプレイヤーが体験すること。

戦闘は、マス目移動よりも
**拠点と拠点をつなぐ Lightline ネットワーク上の移動・配置・防衛・再接続** が中心になる。

---

# 3. 基本用語

## Core Lantern

フィールドの中心拠点。
プレイヤー側のメインベース。

役割：

* 味方の初期配置地点
* Lightline ネットワークの起点
* 敵に侵食されると危険
* チュートリアル後の実戦マップでは、ここを守りながら周囲を再接続していく

イメージ：

> 街の記憶を再点灯するための中心灯台。

---

## Relay Node

中規模拠点。
Core Lantern と Tiny Light をつなぐ中継点。

役割：

* 味方を配置できる
* 接続済みになると行動範囲が広がる
* 敵に占拠されると Lightline が分断される
* 戦略上の主な争奪ポイント

イメージ：

> 通信塔、記憶の交差点、街灯の分岐点。

---

## Tiny Light

小規模拠点。
街の記憶・人々の小さな願い・失われた名前の欠片。

役割：

* 接続するとスコア／報酬／イベントが発生
* 基本的には味方を常駐配置できない、または一部ユニットのみ配置可能
* 孤立している Tiny Light を救うことがステージ目的に絡む

イメージ：

> 誰かの名前、落とし物、忘れられた看板、消えかけた窓明かり。

---

## Lightline

ノード同士を結ぶ線。
味方や敵は Lightline に沿って移動する。

役割：

* 移動経路
* 接続状態の可視化
* 戦況そのものを表す
* 敵に断線・汚染されることがある

状態例：

* 点灯：味方側が接続済み
* くすんでいる：未接続
* 赤黒い：敵に侵食中
* 点滅：危険／次ターン敵が進行予定

---

# 4. バトルシステム概要

## ジャンル

**拠点接続型ターン制ストラテジーRPG**

近い感触：

* 大拠点・中拠点・小拠点をつなぐ陣取り
* 聖戦ケルベロス的な「拠点ライン」感
* ただし目的は征服ではなく「再接続」
* SRPGよりも軽く、パズルよりもドラマがある

---

# 5. ターン進行

## 基本ターン構造

1. プレイヤーが味方全員の行動を選ぶ
2. 行動を確定する
3. 味方が一斉に行動する
4. 敵が行動する
5. 接続・侵食・イベント処理
6. 次ターンへ

---

## 味方の移動

基本ルール：

* 味方は1ターンに1つ、隣接する Light Node へ移動できる
* 移動は Lightline に沿って行う
* 未接続ノードにも移動できるが、危険度が高い
* 接続済みライン上では一部ユニットが強化される可能性あり

---

## 敵の移動

基本ルール：

* 敵も1ターンに1つ、Light Node を移動できる
* 敵は拠点上にばらけて配置される
* 敵は孤立ノードや Relay Node を狙う
* 一部の敵は Lightline を汚染・切断する

---

# 6. 味方ユニット

チュートリアル後の段階では、仲間がある程度集まっている想定。

## 主人公AI：仮称「Little Core」

役割：

* 中心的な修復ユニット
* Core Lantern との相性が高い
* Reconnect 行動の基準になる
* ストーリー上の視点キャラ

性能イメージ：

* 攻撃力は中程度
* 接続・修復能力が高い
* 重要ノードにいると周囲の味方を支援

---

## 仲間ユニットの方向性

現段階では細かい数値より、役割差を重視する。

### 1. Scout系

役割：

* 先行探索
* 遠い Tiny Light の回収
* 危険地帯の確認

特徴：

* 移動に強い
* 戦闘は弱め
* 孤立ノードに向かわせやすい

---

### 2. Guard系

役割：

* Relay Node 防衛
* 敵の進行阻止
* Core Lantern 周辺の守り

特徴：

* 耐久が高い
* 移動は普通
* 敵に狙われても踏みとどまれる

---

### 3. Linker系

役割：

* Lightline の修復
* 孤立ノードの再接続
* 接続ボーナス発生

特徴：

* 直接戦闘は弱め
* ステージ攻略の要
* Tiny Light との相性が高い

---

### 4. Spark系

役割：

* 敵への干渉
* 汚染解除
* 一時的なライン強化

特徴：

* トリッキー
* 敵の行動を遅らせる
* 重要局面で使うと気持ちいい

---

# 7. 敵ユニット

敵は単なるモンスターではなく、
**忘却・断線・ノイズ・なりすまし・無関心** のような概念を持つ存在。

## 基本敵：Noise Bit

役割：

* 小型の敵
* Tiny Light や未接続ノードに群がる
* 序盤の基本敵

特徴：

* 弱い
* 数で押してくる
* 放置すると Lightline をくすませる

---

## 中型敵：Null Crawler

役割：

* Relay Node を狙う
* 接続を切る
* 防衛の練習相手

特徴：

* 少し硬い
* 1体でも放置すると危険
* Guard系の出番を作る

---

## 妨害敵：Mask Echo

役割：

* 名前や記憶を曖昧にする
* Tiny Light の救出を遅らせる
* ストーリー性のある敵

特徴：

* 直接攻撃より妨害中心
* 「名前をなくした街」と相性が良い
* 倒すと住民の記憶イベントが発生しやすい

---

# 8. Re-Link Encounter

## 概要

**Re-Link Encounter** は、敵との通常戦闘というより、
**切れたネットワークをつなぎ直すための局地戦**。

フィールド上で敵と味方が同じノード、または隣接ノードで衝突したときに発生する。

---

## Re-Link Encounter の目的

敵を全滅させるだけでなく、以下のような目的を持つ：

* ノードを奪還する
* Lightline を再点灯する
* Tiny Light を救出する
* Relay Node の侵食を止める
* 敵の進行ルートを遮断する

---

## Encounter の処理案

現段階では、複雑な個別戦闘画面にはせず、
**フィールド上で軽く解決する方式** が合っている。

例：

1. 味方と敵が同じノードに入る
2. ユニットの役割・相性・接続状態で結果判定
3. 短い演出
4. 敵を押し戻す／消す／弱体化する
5. ノード状態が変化する

---

## 接続状態による戦闘補正

接続済みノードでは味方が有利。

例：

* 味方の耐久アップ
* Reconnect 成功率アップ
* 敵の侵食速度ダウン
* 周囲の Tiny Light から支援演出が出る

未接続・孤立ノードではリスクが高い。

例：

* 敵が強くなる
* 味方が押し戻されやすい
* 妨害効果を受けやすい

---

# 9. ステージ仕様

## Reconnect Field 01：名前をなくした街

## ステージテーマ

かつて人々が暮らしていた街。
しかし今は、看板の文字、住民の名前、通りの意味が失われている。

街灯は消え、道はあるのに、どこへ続いていたのか誰も思い出せない。

プレイヤーは Core Lantern を起点に、Relay Node と Tiny Light を再接続し、
街に「名前」を取り戻していく。

---

## ステージの位置づけ

* 実戦チュートリアル後の最初の本格マップ
* Reconnect Field の基本を学ぶ
* 複数ユニットの配置・移動を試す
* 敵の侵食と防衛を理解する
* 「つなぐこと」が勝利条件になると伝える

---

# 10. マップ構成

## 全体構造

マップはやや広め。
開始時に Reconnect Field 全体を見渡せる。

構成：

* 中央付近に Core Lantern
* 周囲に 3〜4個の Relay Node
* 外周や脇道に Tiny Light
* 一部の Lightline は最初から接続済み
* 一部の Tiny Light は孤立している
* 敵は複数の拠点上にばらけている

---

## マップイメージ

```
        [Tiny Light A]       [Tiny Light B]
              \                 /
          [Relay Node 1] -- [Relay Node 2]
                \             /
                 \           /
                [Core Lantern]
                 /           \
          [Relay Node 3] -- [Relay Node 4]
              /                 \
      [Tiny Light C]       [Tiny Light D]
```

ただし、実際には左右対称にしすぎず、
「どこから救うか迷う」形にする。

---

# 11. 初期状態

## 味方

Core Lantern に主人公AIと仲間数名がいる。

想定：

* 主人公AI
* Scout系 1体
* Guard系 1体
* Linker系 1体
* Spark系 1体

合計 4〜5体程度。

## 敵

敵は以下のように配置。

* Relay Node 2 に Null Crawler
* Relay Node 4 付近に Noise Bit 複数
* Tiny Light B 周辺に Mask Echo
* 外周に Noise Bit が点在

---

## ノード状態

開始時：

* Core Lantern：点灯済み
* Relay Node 1：接続済み
* Relay Node 3：薄く接続済み
* Relay Node 2：敵に侵食中
* Relay Node 4：未接続
* Tiny Light A：救出可能
* Tiny Light B：名前を失っている
* Tiny Light C：孤立
* Tiny Light D：敵が接近中

---

# 12. 勝利条件

メイン勝利条件：

**街の主要 Lightline を再接続し、Core Lantern から3つ以上の Relay Node を点灯させる。**

追加条件：

* Tiny Light を一定数救出する
* Mask Echo を退ける
* Core Lantern を守り切る

---

## 推奨勝利条件案

ステージ01では、わかりやすくするために：

**勝利条件：**

* Relay Node を3つ再接続する
* Tiny Light を2つ救出する

**敗北条件：**

* Core Lantern が侵食される
* 味方全員が行動不能になる
* 一定ターン経過で街の記憶が崩壊する、は今回はまだ入れなくてもよい

ターン制限は、最初の本格マップでは緩めか、なしでも良い。
プレイヤーに探索と接続の気持ちよさを味わわせる方が大事。

---

# 13. ステージ進行

## フェーズ1：見渡す

開始時にマップ全体を見せる。

プレイヤーが理解すること：

* 拠点が点在している
* 線でつながっている
* 敵もノード上にいる
* 孤立した Tiny Light がある
* どこから助けるか選べる

演出：

> Core Lantern が弱く光り、周囲の消えた街灯がぼんやり浮かぶ。

---

## フェーズ2：最初の再接続

Relay Node 1 または 3 に向かう。

学ぶこと：

* 味方は1ターンに1ノード移動
* 接続済みライン上では安全
* Relay Node に味方を置くとネットワークが安定する

---

## フェーズ3：敵との衝突

Relay Node 2 または 4 付近で敵と接触。

学ぶこと：

* 敵もノードを移動する
* 放置すると重要拠点が侵食される
* Guard や Spark の使いどころがある

---

## フェーズ4：Tiny Light の救出

孤立した Tiny Light に Linker または Scout を向かわせる。

学ぶこと：

* 戦闘だけでは勝てない
* 小さな光を救うことが攻略に必要
* 救出すると街の記憶が少し戻る

---

## フェーズ5：名前の回復

Mask Echo を退けるか、Tiny Light B を救出すると、街の一部の名前が戻る。

演出例：

* 「■■通り」だった看板が「すずらん通り」になる
* 「だれかの家」が「ミナの家」になる
* マップ上のノード名が変化する

---

## フェーズ6：クリア

3つの Relay Node が Core Lantern とつながると、街全体に光が流れる。

クリア演出：

* Lightline が一斉に点灯
* Tiny Light がふわっと舞う
* 街の名前が少し戻る
* でもまだ遠くの区画は暗いまま

次ステージへの引き：

> 「この街は、まだ全部を思い出したわけじゃない。」

---

# 14. プレイヤーに学ばせたいこと

Reconnect Field 01 で学ぶべきことは以下。

1. 拠点をつなぐことが基本
2. 味方はノード上を移動する
3. 敵もノードを移動する
4. Relay Node は重要
5. Tiny Light は救出対象
6. 接続済みラインは安全
7. 孤立ノードにはリスクがある
8. 戦闘と接続の両方が必要
9. 配置順と移動先の選択が面白さになる

# 15. UI仕様

* Core Lantern

* 敵ユニット

## ノード表示

ノードには状態を色やエフェクトで表示。

例：

* 青白い光：味方接続済み
* 灰色：未接続
* 赤黒いノイズ：敵侵食中
* 黄色い点滅：救出可能な Tiny Light
* 白い強光：重要目標

---

## ユニット選択

ユニットを選ぶと：

* 移動可能な隣接ノードが光る
* 推奨行動が軽く表示される
* 危険なノードには警告が出る

例：

> 「このノードは孤立しています。移動すると支援を受けられません。」

---

## 行動確定

全員の行動を選んでから「Turn Execute」。

行動前に以下を確認できる：

* 誰がどこへ移動するか
* 敵の予想進行
* 接続される予定のライン
* 危険な衝突ポイント

# 17. 演出仕様

## Lightline 再接続演出

* ノード間に細い光が走る
* 一瞬だけ音が鳴る
* 周囲のノイズが消える
* Tiny Light が反応する

---

## Tiny Light 救出演出

Tiny Light に触れると短いテキストが出る。

例：

> 「……わたし、ここにいた気がする。」
> 「名前はまだ思い出せない。でも、あたたかい。」

救出後、Tiny Light は Core Lantern 側へ小さく光を送る。

---

## 敵侵食演出

敵がノードを侵食すると：

* Lightline が赤黒くにじむ
* ノード名が文字化けする
* 周囲の Tiny Light が震える

# 23. 面白さの核

現段階で一番強いのはここ。

**敵を倒すゲームではなく、
「どこを先につなぐか」「誰をどこに置くか」「孤立した光を助けるか、防衛を優先するか」で悩むゲーム。**

プレイヤーの選択が、マップ上の光の形として見える。

良いプレイをすると：

* 線がつながる
* 街が明るくなる
* 名前が戻る
* 敵のノイズが消える
* 仲間が役割を果たした感じが出る

この体験が Little Lights Protocol の中心。

* 正確なダメージ計算

* キャラの正式名
* 成長システム
* スキルツリー
* 恒久的な仲間ロストの有無
* Unity / Godot / Web の最終選定
* 長編ストーリーの全体構成

まずは、
**Reconnect Field 01 が遊んでいて気持ちいいか**
を検証するのが最優先。

# 25. 次に作るべきもの

次の開発ステップはこの順番がよさそう。

## Step 1：紙プロトタイプ

ノードと線だけで遊べるか確認。

確認すること：

* 1ターン1ノード移動が面白いか
* 敵の圧がちょうどいいか
* Relay Node の取り合いが楽しいか
* Tiny Light 救出に意味があるか

---

## Step 2：Web / Godot の簡易プロトタイプ

最初は見た目より、操作とルール確認。

必要なもの：

* ノードクリック
* 移動予約
* ターン実行
* 敵移動
* 接続状態変化
* 勝利判定

---

## Step 3：演出追加

ルールが面白ければ、光の演出を足す。

追加するもの：

* Lightline 点灯
* Tiny Light 救出
* 文字化け解除
* Core Lantern の発光
* 敵侵食のにじみ

---

## Step 4：キャラ性追加

最後に、仲間たちのセリフや個性を足す。

例：

* Scout「先に見てくるね！」
* Guard「ここは任せて」
* Linker「線が……まだ生きてる」
* Spark「ノイズ、ちょっと黙ってて！」

# 26. 現段階の結論

**Little Lights Protocol は、かなり良い方向に育っている。**

特に強いのは、

* Core Lantern
* Relay Node
* Tiny Light
* Lightline
* Re-Link Encounter
* 名前をなくした街

この6つが、世界観とゲームシステムを自然につないでいるところ。

このまま進めるなら、最初のMVPは：

**「名前をなくした街で、仲間をノードに移動させ、Relay Node を再接続し、Tiny Light を救うターン制マップ」**

として作るのが一番いい。

かなり遊べる芯が見えてきた。
