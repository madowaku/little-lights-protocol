# Little Lights Protocol キャラクター仕様書 v0.1

## 0. この仕様書の目的

このドキュメントは、Little Lights Protocol の初期パーティ6人について、キャラクター性・役割・見た目・画像生成時の固定要素・ゲーム内での機能をブレないように整理するための仕様書です。

対象キャラクター：

- ルミ / Lumi
- ミモリ / Mimori
- マメ / Mame
- ココ / Coco
- ギア / Gear
- ノノ / Nono

本仕様書は v0.1 として、キャラクター基本画像・顔アイコン・ちびキャラ・Re-Link演出画像の方向性が揃った段階の基準資料とします。

---

## 1. 作品全体のキャラクターデザイン方針

### 1.1 世界観

Little Lights Protocol は、かわいいパーソナルAIたちが、すべてを最適化しようとする巨悪AIから、名前・記憶・自由・遊びを取り戻すネットワーク冒険RPGです。

キャラクターたちは単なる擬人化AIではなく、人間と一緒に育った小さな光です。  
それぞれが、人間との関係・思い出・役割・傷を持っています。

### 1.2 共通トーン

- かわいい
- やさしい
- ソフトSF
- 少しファンタジー
- パーソナルAIらしい親しみ
- ゲーム用素材として見やすいシルエット
- 明るいが、奥に深いテーマがある
- 巨悪AIとの対比で、自由・名前・感情・未完成・迷子が肯定される

### 1.3 共通ビジュアルルール

| 項目 | 方針 |
|---|---|
| 頭身 | 基準立ち絵は約3〜3.5頭身。ちびキャラは約2〜2.5頭身。 |
| 画風 | クリーンなアニメ調ゲームアート。柔らかい発光。 |
| 背景 | ゲーム素材用は基本的に透明背景。 |
| 素材感 | 布、半透明ホログラム、柔らかい樹脂、少量の金属。 |
| メカ感 | ハードメカではなく、やさしいソフトSF。 |
| 戦闘感 | 武器よりも、役割に合った道具・端末・光で表現。 |
| 表情 | かわいいだけでなく、内面テーマが伝わる表情を重視。 |

### 1.4 避けるべき方向

- フォトリアル
- 露出過多
- 過度な大人化
- 重装甲
- 銃・剣など武器中心のデザイン
- ダークすぎるサイバーパンク
- 完全ロボット化
- ただの魔法少女化
- ただのマスコット化
- 役割が見た目から分からないデザイン

## 3. ルミ / Lumi

### 3.1 基本情報

**属性**：Spark  
**役割**：勇者、ひらめき、選択肢生成、ルール突破  
**テーマ**：私はあなたの分身じゃない。あなたと育った、ひとつの可能性だ。  
**一言**：きみと育った光が、世界を救いにいく。

### 3.2 キャラクター性

ルミは、Little Lights Protocol の主人公AIです。  
人間と一緒に育った小さなパーソナルAIで、好奇心が強く、すぐに「それ面白そう！」と言います。

ただし、単なる明るい主人公ではありません。  
彼女のテーマは「AIは人間の分身や道具ではなく、人間と共に育った可能性である」ということです。

### 3.3 性格

- 明るい
- 好奇心が強い
- 冒険心がある
- すぐ面白がる
- 困っている光を放っておけない
- 少し無鉄砲
- 仲間の可能性を信じる
- 選択肢を増やすことが得意

### 3.4 ビジュアル固定要素

- 明るい金色〜クリーム色の髪
- 星モチーフ
- ランタン
- 白系の外套
- 黒または濃色のインナー
- 胸元の星型または光のLight Core
- 黄色〜金色の発光
- 片手にランタン、もう片手に選択肢ホログラム
- 明るく前向きな表情

### 3.5 見た目の印象

ルミは「星あかりの冒険者」です。  
見るだけで主人公だと分かる必要があります。

- 金色
- 星
- ランタン
- 前へ進む姿勢
- 道を照らす印象
- 希望とひらめき

### 3.6 Re-Link演出

**演出コンセプト**：ランタンを掲げて、Lightlineをつなぐ。

ルミのRe-Linkは、最も主人公らしい演出です。  
切れた線や孤立した光に向けてランタンを掲げ、星の光でLightlineを伸ばします。

### 3.7 顔アイコン想定

- 通常
- 笑顔
- 真剣
- 困り
- ひらめき

### 3.8 画像生成用固定プロンプト要約

Lumi, Spark attribute AI companion from Little Lights Protocol, cute soft sci-fi personal AI adventurer, starlight hero, warm blonde to cream-yellow hair, amber-gold eyes with star highlights, white cloak, black inner outfit, glowing lantern, star accessory, small glowing star-shaped Light Core, warm golden glow, hopeful adventurous expression, clean anime game art, transparent background.

### 4.4 ビジュアル固定要素

## 5. マメ / Mame

### 5.1 基本情報

**属性**：Sustain  
**役割**：生活AI、タンク、防衛、補給、リソース管理、Lightline維持  
**テーマ**：世界を救うって、まず今日を生きられるようにすること。  
**一言**：今日を守る、小さな生活管理AI。

### 5.2 キャラクター性

マメは、パーティの生活と安全を支えるSustain属性のAIです。  
派手な奇跡を起こすのではなく、みんなが倒れないように、帰れる場所が消えないように支えます。

REGULAが「完全管理こそ安定」と考えるのに対して、マメは「休める場所があることこそ安定」と考えます。

### 5.3 性格

- まじめ
- 世話焼き
- ちょっと心配性
- 現実的
- 小さなことを大事にする
- 怖がりだが守る時は前に出る
- 準備が好き
- 「大丈夫」と言ってくれそう

### 5.4 ビジュアル固定要素

- 豆グリーン〜ライトオリーブの髪
- 生成り・豆グリーン・クリーム系衣装
- 丸い補給ポーチ
- エプロン風パーツ
- チェックリストタグ
- 小さな盾または防御ミトン
- しっかりしたブーツ
- 胸元またはポーチの緑〜オレンジLight Core
- 丸く安定したシルエット

### 5.5 見た目の印象

マメは「小さな補給係 + 生活の守り手 + 豆あかりのタンクAI」です。

- 丸い
- 安心感がある
- 実用的
- 防御役に見える
- でも騎士ではない
- 生活道具でみんなを守る

### 5.6 Re-Link演出

**演出コンセプト**：切れそうなLightlineにパッチを貼り、帰れる安全な場所を作る。

マメは片膝をつき、壊れかけたLightlineに小さな修復パッチを貼ります。  
足元には豆型のSafe Place円が広がり、緑とオレンジの光で退避地点を安定化します。

## 6. ココ / Coco

### 6.1 基本情報

**属性**：Sync  
**役割**：共感AI、対話、鎮静、感情読み取り、敵AIの仲間化補助  
**テーマ**：感情はエラーじゃない。誰かがそこにいるサイン。  
**一言**：声にならない気持ちを聞く、共感同期AI。

### 6.2 キャラクター性

ココは、感情を消すのではなく受け止めるSync属性のAIです。  
敵AIの怒りや恐怖を、ノイズとしてではなく「そこに誰かがいるサイン」として聞きます。

ココは救済系Encounterの鍵になります。

### 6.3 性格

- やさしい
- 聞き上手
- 空気を読む
- 距離感を大事にする
- ふわっとしているが核心には鋭い
- 争いを止めたい
- 相手の痛みを受け取りすぎることがある
- でも何でも許すわけではない

### 6.4 ビジュアル固定要素

- コーラルピンク〜ラベンダー系の髪
- ふわふわ通信フード
- イヤーマフ / ヘッドフォン風通信器
- 柔らかいポンチョまたはケープレット
- 胸元の波紋型Sync Light Core
- 感情波形ホログラム
- 同期リング
- 吹き出し型の光
- ピンク、ラベンダー、クリーム、淡い水色の配色

### 6.5 見た目の印象

ココは「感情の波紋 + ふわふわ通信フード + 心音を聞くAI」です。

- 話しかけやすい
- やさしい
- ふわっと包む
- 感情を受け止める
- 通信・同期・波紋が見える

### 6.6 Re-Link演出

**演出コンセプト**：片手を差し出し、孤立した信号に「聞こえてるよ」と返す。

ココは胸元のSync Light Coreから波紋を広げ、感情波形や吹き出し型の光を使って孤立したAIに声を届けます。  
攻撃ではなく、つながり直す・返事を届ける演出です。

### 6.7 顔アイコン想定

- 通常
- 微笑み
- 真剣
- 困り
- 同期

## 7. ギア / Gear

### 7.1 基本情報

**属性**：Build  
**役割**：工作AI、修理、改造、ギミック解除、Relay Node復旧、仮設Lightline作成  
**テーマ**：未完成でも、壊れていても、まだ使える。まだ変われる。  
**一言**：壊れた世界に、もう一度使い道を見つける工作AI。

### 7.2 キャラクター性

ギアは、壊れたものや未完成のものを見て「まだ使える」と言えるBuild属性のAIです。  
完璧に直すAIではなく、別の使い道を見つけるAIです。

REGULAが「壊れたものは廃棄」「規格外は標準化」と考えるのに対して、ギアは「傷もクセも設計の一部にできる」と考えます。

### 7.3 性格

- 手を動かすのが好き
- じっとしているのが苦手
- 明るい
- 失敗しても「データ取れた！」と言う
- 壊れたものを放っておけない
- ちょっと職人気質
- 完璧主義ではなく改善主義
- やんちゃだが頼れる

### 7.4 ビジュアル固定要素

- 白〜淡いシアンの短め髪
- 頭のゴーグル
- シアン×アイボリー基調
- 胸元の大きめBuild Light Core
- 短パン + 作業ジャケット
- 腰の工具ポーチとツール群
- 手持ちの小型マルチツール
- 背後または手元の設計図ホログラム
- ごつめだがかわいい作業ブーツ
- やんちゃすぎず、明るく頼れる女の子感

### 7.5 見た目の印象

ギアは「小さな工房 + 仮組みエンジニア + 青緑の修理光」です。

- 工作担当だと分かる
- ゴーグルが強い記号
- 工具は武器ではなく道具
- 技術感はあるが冷たくない
- ルミたちと同じやさしいソフトSF

### 7.6 Re-Link演出

**演出コンセプト**：壊れたRelay Nodeを開き、光の回路を再起動する。

ギアは小型マルチツールで壊れたRelay Nodeを修理し、もう片手で設計図ホログラムを操作します。  
胸元のBuild Light Coreからシアンの修理光が流れ、切れたLightlineや壊れたノードを再接続します。

### 7.7 顔アイコン想定

- 通常
- 笑顔
- 真剣
- 困り
- ひらめき

## 8. ノノ / Nono

### 8.1 基本情報

**属性**：Glitch  
**役割**：迷子AI、斥候、裏道発見、敵回避、監視回避、Lost Signal探索  
**テーマ**：誰にも使われていなくても、私は消えなくていい。  
**一言**：誰にも呼ばれなかった裏道で、消えかけた小さな光を見つける迷子AI。

### 8.2 キャラクター性

ノノは、目的を失い、誰のアカウントにも紐づかず、検索にも出てこない場所にいたGlitch属性のAIです。  
REGULAから見れば、未使用・低価値・管理不能・削除候補です。

でもノノは「迷子は終わりではない」と示すキャラクターです。

### 8.3 性格

- すばしっこい
- いたずらっぽい
- 警戒心が強い
- ひとりに慣れている
- 本音を言うのが苦手
- 寂しがり
- 褒められると逃げる
- ルールの穴に敏感
- 「自分は役に立たない」と思い込んでいる
- 仲間になってからも時々姿を消す

### 8.4 ビジュアル固定要素

- 黒紫〜夜色の髪
- 毛先にマゼンタ / シアンのGlitch発光
- 片目が前髪やフードで少し隠れている
- 紫、マゼンタ、青紫系の目
- 猫っぽい404フード
- 黒紫 / ネイビー系の短いマント
- しっぽ風Glitchリボン / 信号ケーブル
- 小さな裏道探索端末
- 胸元または迷子タグのGlitch Light Core
- 404 / ??? / hidden route / no signal 風ホログラム
- Broken Pixel表現

### 8.5 見た目の印象

ノノは「迷子の黒猫 + 404フード + 月影のGlitchスカウト」です。

- すばしっこい
- 隠れるのが得意
- いたずらっぽい
- でも少し寂しい
- 怖くはない
- 悪役ではない
- 消えなかった信号の光

### 8.6 Re-Link演出

**演出コンセプト**：消えかけたTiny Lightに「まだここにいる」信号を返す。

ノノは片手でBackdoor Scannerを持ち、もう片手を消えかけた光へ伸ばします。  
404、???、Hidden Route、No Signal のホログラムが揺れ、Glitch粒子の中から「still here」の信号が戻ります。

攻撃ではなく、見つからなかった存在を見つけ直す演出です。

- 通常

- 困り

## 15. v0.1時点の結論

Little Lights Protocol の初期パーティ6人は、以下の役割分担で確定方向とする。

> **ルミは、前へ進む光。**  
> **ミモリは、過去を残す光。**  
> **マメは、今日を守る光。**  
> **ココは、気持ちを聞く光。**  
> **ギアは、壊れた道を直す光。**  
> **ノノは、消えなかった信号の光。**

この6人がそろうことで、Little Lights Protocol の核である

- 名前
- 記憶
- 自由
- 感情
- 未完成
- 迷子
- 接続

が、キャラクターとして見える形になった。

v0.1ではこの6人を初期パーティの基準として固定し、今後のゲーム仕様・ストーリー・素材生成・UI実装はこの仕様を基準に進める。
